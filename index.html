Overview

A local-first, no-code data science desktop app. Users upload a CSV and move through the full workflow: statistics, visualization, cleaning, feature selection, scaling, training scikit-learn models, comparing scores, and making predictions, all without writing Python. Entire projects save to a portable file that can be reopened later.

My role

Sole builder. I owned the product end to end, including the Streamlit UI and workflow, the local data pipeline, the ML orchestration, the custom project format, and desktop packaging for macOS and Windows.

The problem

Basic data science often means stitching pandas and scikit-learn together by hand in notebooks, which leaves out analysts and students who understand the data but do not write code. DataTellsDS brings the whole tabular workflow, from import and exploration to preprocessing, training, and prediction, into one guided app that runs entirely on the user's machine.

Architecture
PythonStreamlitpandasscikit-learnPlotlyPyInstaller
  • Streamlit UI drives the full workflow: upload, statistics, EDA, data prep, training, and prediction.
  • File-based project state: datasets, encodings, scalers, trained models, and config persist locally as CSV, JSON, and pickle files.
  • Separate execution for preprocessing and model training, orchestrated from the UI.
  • Custom .dtds format: a zip archive that bundles a full session for export and reopening.
  • PyInstaller packaging into desktop installers for macOS (Apple Silicon and Intel) and Windows.
Engineering highlights
  • Metadata-driven model config: dozens of scikit-learn estimators (34 classifiers, 51 regressors) exposed through JSON parameter schemas, so users tune models through forms instead of code.
  • End-to-end tabular pipeline: null imputation (mean, median, mode), categorical encoding, feature selection (VarianceThreshold, SelectKBest), and scaling (Standard, MinMax, Robust), all driven from the UI.
  • Generated prediction forms: builds an input form from the retained features, applies the saved encodings and scaler, and runs inference on the chosen model.
  • Project portability: designed the .dtds bundle to package data, preprocessing artifacts, trained models, and workflow state into one reopenable file.
  • Local-first by design: everything runs on the user's machine, so no data leaves the device for analysis or training.
Outcome

Shipped a full product from zero to one with cross-platform desktop installers, covering the entire tabular data science workflow in one local app.

Overview

An open-source contribution to Apache Spark's Python API (PySpark) that automates the tedious step of making data model-ready by imputing missing values and encoding categorical columns in a single line using the Pandas API on Spark. It also includes tested integration for AWS S3 through boto3 and for TensorFlow workflows.

My role

Author and open-source contributor. I designed the feature set, built it against the Spark source, and wrote the tests and documentation.

The problem

Before a model can train, every column has to be numeric and complete. In Spark that usually means repetitive preprocessing, with null handling and categorical encoding written differently in every project, all while still needing to scale to large datasets.

Approach
  • Automatic imputation: fills missing values across the dataset with multiple selectable strategies.
  • Automatic encoding: handles label, count, and one-hot encoding for categorical and boolean columns.
  • Built on the Pandas API on Spark, so it feels familiar but runs distributed.
  • Cloud + ML integration: uses boto3 to read and write files from AWS S3, with TensorFlow model execution tested inside the Spark environment.
Engineering highlights
  • One-line API: collapses an entire preprocessing block into a single call for imputation followed by encoding.
  • Built against the Spark source: cloned and extended the framework, then validated on small datasets before scaling to large ones.
  • Hardened with multiple test cases including exception paths, with inline documentation for every method.
Outcome

Open-sourced and documented so others can prepare data, work with S3, and run TensorFlow models from one place, continuing the same research line as my published work.
